Removes a key/value pair from a GLib.Tree.

If the GLib.Tree was created using g_tree_new_full (not introspectable), the key and value are freed using the supplied destroy functions, otherwise you have to make sure that any dynamically allocated values are freed yourself. If the key does not exist in the GLib.Tree, the function does nothing.

function GLib.Tree.prototype.destroy(): {
    // javascript wrapper for 'g_tree_destroy'

Removes all keys and values from the GLib.Tree and decreases its reference count by one. If keys and/or values are dynamically allocated, you should either free them first or create the GLib.Tree using g_tree_new_full (not introspectable). In the latter case the destroy functions you supplied will be called on all keys and values before destroying the GLib.Tree.

GLib.Tree.prototype.insert 

function GLib.Tree.prototype.insert(key: Object, value: Object): {
    // javascript wrapper for 'g_tree_insert'
}

Inserts a key/value pair into a GLib.Tree.

If the given key already exists in the GLib.Tree its corresponding value is set to the new value. If you supplied a value_destroy_func when creating the GLib.Tree, the old value is freed using that function. If you supplied a key_destroy_func when creating the GLib.Tree, the passed key is freed using that function.

The tree is automatically 'balanced' as new key/value pairs are added, so that the distance from the root to every leaf is as small as possible.

GLib.Tree.prototype.replace 

function GLib.Tree.prototype.replace(key: Object, value: Object): {
    // javascript wrapper for 'g_tree_replace'
}

Inserts a new key and value into a GLib.Tree similar to GLib.Tree.prototype.insert. The difference is that if the key already exists in the GLib.Tree, it gets replaced by the new key. If you supplied a value_destroy_func when creating the GLib.Tree, the old value is freed using that function. If you supplied a key_destroy_func when creating the GLib.Tree, the old key is freed using that function.

The tree is automatically 'balanced' as new key/value pairs are added, so that the distance from the root to every leaf is as small as possible.

GLib.Tree.prototype.unref 

function GLib.Tree.prototype.unref(): {
    // javascript wrapper for 'g_tree_unref'
}

Decrements the reference count of tree by one. If the reference count drops to 0, all keys and values will be destroyed (if destroy functions were specified) and all memory allocated by tree will be released.

It is safe to call this function from any thread.

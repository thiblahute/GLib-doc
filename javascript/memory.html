GLib.prototype.mem_is_system_malloc 

function GLib.prototype.mem_is_system_malloc(): {
    // javascript wrapper for 'g_mem_is_system_malloc'
}

Checks whether the allocator used by GLib.prototype.malloc is the system's malloc implementation. If it returns true memory allocated with malloc can be used interchangeable with memory allocated using GLib.prototype.malloc. This function is useful for avoiding an extra copy of allocated memory returned by a non-GLib-based API.

Returns:

if true, malloc and GLib.prototype.malloc can be mixed.

 Number

deprecated : 2.46: GLib always uses the system malloc, so this function always returns %TRUE.

GLib.prototype.realloc 

function GLib.prototype.realloc(mem: Object, n_bytes: Number): {
    // javascript wrapper for 'g_realloc'
}

Reallocates the memory pointed to by mem, so that it now has space for n_bytes bytes of memory. It returns the new address of the memory, which may have been moved. mem may be null, in which case it's considered to have zero-length. n_bytes may be 0, in which case null will be returned and mem will be freed unless it is null.

Parameters

mem:

the memory to reallocate

 Object

n_bytes:

new size of the memory in bytes

 Number

Returns:

the new address of the allocated memory

Structures

GLib.MemVTable

A set of functions used to perform memory allocation. The same GLib.MemVTable must be used for all allocations in the same program; a call to GLib.prototype.mem_set_vtable, if it exists, should be prior to any use of GLib.

This functions related to this has been deprecated in 2.46, and no longer work.

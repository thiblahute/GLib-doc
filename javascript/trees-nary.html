Structures

GLib.Node

The GLib.Node struct represents one node in a [n-ary tree][glib-N-ary-Trees].

Attributes

gpointer data:

contains the actual data of the node.

GNode * next:

points to the node's next sibling (a sibling is another GNode with the same parent).

GNode * prev:

points to the node's previous sibling.

GNode * parent:

points to the parent of the GNode, or is NULL if the GNode is the root of the tree.

GNode * children:

points to the first child of the GNode. The other children are accessed by using the next pointer of each child.

Enumerations

enum GLib.TraverseType

Specifies the type of traveral performed by g_tree_traverse (not introspectable), g_node_traverse (not introspectable) and g_node_find (not introspectable). The different orders are illustrated here:

  • In order: A, B, C, D, E, F, G, H, I
  • Pre order: F, B, A, D, C, E, G, I, H
  • Post order: A, C, E, D, B, H, I, G, F
  • Level order: F, B, G, A, D, I, C, E, H

Members

GLib.TraverseType.prototype.in_order

vists a node's left child first, then the node itself, then its right child. This is the one to use if you want the output sorted according to the compare function.

 Value: 0

GLib.TraverseType.prototype.pre_order

visits a node, then its children.

 Value: 1

GLib.TraverseType.prototype.post_order

visits the node's children, then the node itself.

 Value: 2

GLib.TraverseType.prototype.level_order

is not implemented for [balanced binary trees][glib-Balanced-Binary-Trees]. For [n-ary trees][glib-N-ary-Trees], it vists the root node first, then its children, then its grandchildren, and so on. Note that this is less efficient than the other orders.

 Value: 3

enum GLib.TraverseFlags

Specifies which nodes are visited during several of the tree functions, including g_node_traverse (not introspectable) and g_node_find (not introspectable).

Members

GLib.TraverseFlags.prototype.leaves

only leaf nodes should be visited. This name has been introduced in 2.6, for older version use GLib.TraverseFlags.prototype.leafs.

 Value: 1

GLib.TraverseFlags.prototype.non_leaves

only non-leaf nodes should be visited. This name has been introduced in 2.6, for older version use GLib.TraverseFlags.prototype.non_leafs.

 Value: 2

GLib.TraverseFlags.prototype.all

all nodes should be visited.

 Value: 3

GLib.TraverseFlags.prototype.mask

a mask of all traverse flags.

 Value: 3

GLib.TraverseFlags.prototype.leafs

identical to GLib.TraverseFlags.prototype.leaves.

 Value: 1

GLib.TraverseFlags.prototype.non_leafs
